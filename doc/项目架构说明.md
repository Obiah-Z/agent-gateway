# AI Agent Gateway 项目架构说明

## 1. 项目根目录

项目根目录：`/home/obiah/Desktop/claw0/gateway`

```text
gateway/
  agent_gateway/      核心源码
  config/             静态配置
  workspace/          Prompt、记忆、技能、Cron 等运行工作区
  data/               运行数据目录
  tests/              自动化测试
  doc/                补充文档
```

### 根目录主要文件

- `README.md`
  项目总说明文档。

- `pyproject.toml`
  Python 构建配置，定义依赖、入口命令和静态资源打包规则。

- `.env.example`
  环境变量模板。

- `.env`
  本地实际运行配置文件。

- `PROJECT_PLAN.md`
  项目阶段规划文档。

## 3. `runtime/` 运行内核

```text
runtime/
  domain/
  execution/
  state/
  observability/
```

### `runtime/domain/`

放核心领域对象与路由基础。

- `domain/models.py`
  定义核心模型，例如 `AgentConfig`、`InboundMessage`、`OutboundMessage`、`Binding`、`RouteResolution`、`AgentReply`。

- `domain/ids.py`
  内部标识规范化逻辑。

- `domain/agents.py`
  `AgentManager`，维护运行时 Agent 注册表。

- `domain/router.py`
  路由规则匹配与 session key 生成。

### `runtime/execution/`

放执行链路与运行时服务。

- `execution/loop.py`
  Agent 单轮执行闭环。

- `execution/resilience.py`
  模型调用、profile 切换、tool calling 闭环与容灾。

- `execution/dispatcher.py`
  入站消息分发器，负责路由、执行、回复入队。

- `execution/channel_runtime.py`
  通道运行时，接收入站事件并交给 Dispatcher。

- `execution/autonomy.py`
  Heartbeat、全局 Cron、Agent 局部 Cron、新闻简报等主动任务。

- `execution/delivery_runtime.py`
  后台可靠投递执行器。

- `execution/control_plane.py`
  控制面对象，为 Dashboard 和 WebSocket API 提供状态数据。

- `execution/metrics_runtime.py`
  指标采集运行时。

- `execution/alerts_runtime.py`
  告警计算与通知运行时。

- `execution/lanes.py`
  命名并发车道控制。

- `execution/agent_manifest.py`
  Agent manifest 模板与能力校验。

### `runtime/state/`

放持久化状态与队列。

- `state/store.py`
  会话历史 JSONL 存储。

- `state/context.py`
  长上下文保护与压缩逻辑。

- `state/queue.py`
  本地可靠投递队列，管理 pending / failed / retry。

### `runtime/observability/`

放事件、指标、告警能力。

- `observability/events.py`
  统一运行事件模型与 JSONL 存储。

- `observability/metrics.py`
  指标快照模型与读取。

- `observability/alerts.py`
  告警规则、告警状态、告警历史模型。

## 4. `gateways/` 外部接入

```text
gateways/
  messaging/
  feishu/
  control/
```

### `gateways/messaging/`

放通用消息通道基础。

- `messaging/base.py`
  通道抽象基类和账号对象。

- `messaging/manager.py`
  通道管理器。

- `messaging/bootstrap.py`
  根据 `config/channels.json` 构建通道实例。

- `messaging/cli.py`
  CLI 通道。

- `messaging/telegram.py`
  Telegram 通道。

### `gateways/feishu/`

放飞书相关接入与适配。

- `feishu/channel.py`
  飞书消息发送主适配层。

- `feishu/cards.py`
  飞书卡片渲染和发送载荷构造。

- `feishu/state.py`
  飞书卡片状态和通道侧状态。

- `feishu/security.py`
  飞书签名、时间窗、解密与合法性校验。

- `feishu/http.py`
  飞书 Webhook HTTP 接入。

- `feishu/long_connection.py`
  飞书长连接模式接入。

- `feishu/onboarding.py`
  飞书用户绑定与 onboarding 流程。

### `gateways/control/`

- `control/websocket_server.py`
  WebSocket JSON-RPC 控制面服务端。

## 5. `ai/` 智能能力

```text
ai/
  context/
  tools/
  news/
```

### `ai/context/`

放 Prompt、记忆、技能。

- `context/prompt.py`
  Prompt 装配器，从 `workspace/` 组装系统 prompt。

- `context/memory.py`
  长期记忆读取、写入工具、最近记忆查看。

- `context/skills.py`
  发现和加载 `workspace/skills/*/SKILL.md` 中的技能说明。

### `ai/tools/`

放工具系统。

- `tools/registry.py`
  工具注册表。

- `tools/builtin.py`
  内置工具实现，例如 bash、文件读写。

- `tools/web_search.py`
  联网搜索工具接入。

### `ai/news/`

放新闻采集与简报能力。

- `news/models.py`
  新闻模型定义。

- `news/sources.py`
  新闻源解析。

- `news/store.py`
  新闻结果存储。

- `news/collector.py`
  新闻采集器。

- `news/digest.py`
  新闻摘要 prompt 和整理逻辑。
