# AI Agent Gateway 项目目录说明

本文档按目录说明当前项目中的主要文件用途。重点是帮助快速理解“每个目录放什么、每个文件负责什么”，不展开运行链路和架构演进。

项目根目录：`/home/obiah/Desktop/claw0/gateway`

## 1. 根目录

```text
gateway/
  agent_gateway/                 核心 Python 源码
  config/                        Agent、路由、通道、模型 profile 配置
  deploy/                        部署文档和 systemd 配置
  doc/                           项目补充文档、压测清单、技术选型报告
  scripts/                       压测、容量基线、分布式 smoke 脚本
  tests/                         自动化测试
  workspace/                     Prompt、记忆、技能、Cron 和运行报告
  data/                          本地运行数据，通常不作为源码文档重点维护
```

根目录主要文件：

- `.dockerignore`：Docker 构建上下文排除规则。
- `.env`：本机实际运行环境变量，包含模型、中间件、飞书等敏感配置。
- `.env.example`：环境变量模板，用于新环境初始化。
- `.gitignore`：Git 忽略规则。
- `Dockerfile`：Gateway 镜像构建文件。
- `docker-compose.yml`：基础 Compose 配置，包含 Redis、PostgreSQL、RabbitMQ 和单体 `gateway` 服务。
- `docker-compose.roles.yml`：多角色部署覆盖文件，拆分 api、worker、delivery、scheduler、dashboard 等服务。
- `docker-compose.workers.yml`：多 worker 覆盖文件，定义多个 `gateway-worker-*` 实例。
- `PROJECT_PLAN.md`：项目阶段计划和后续任务清单。
- `README.md`：项目总览、快速启动、部署和使用说明。
- `pyproject.toml`：Python 包配置、依赖、命令入口和打包规则。

## 2. `agent_gateway/`

核心源码目录，包含应用装配、配置、AI 能力、通道接入、运行时、状态存储和运维面板。

### 2.1 顶层文件

- `__init__.py`：Python 包初始化文件。
- `app.py`：应用装配入口，创建 settings、通道、Agent、工具、任务队列、worker、投递、控制面、观测等核心对象。
- `config.py`：环境变量和路径配置定义，例如模型配置、中间件地址、端口、队列参数、worker 参数。
- `config_loader.py`：加载和保存 `config/*.json`，为控制面热更新提供配置读写能力。

### 2.4 `agent_gateway/runtime/`

运行内核目录，负责领域模型、执行链路、中间件客户端、观测、状态存储和任务系统。

- `__init__.py`：runtime 子包初始化文件。
- `diagnostics.py`：doctor 检查逻辑，用于检查环境变量、中间件连通性、schema、安全配置等。

#### `agent_gateway/runtime/domain/`

- `__init__.py`：domain 子包初始化文件。
- `agents.py`：Agent 注册表和 Agent 配置管理。
- `ids.py`：Agent ID、通道 ID、session key 等命名规范化工具。
- `models.py`：核心领域模型，例如 `InboundMessage`、`OutboundMessage`、`AgentConfig`、`Binding`、`AgentReply`。
- `router.py`：入站路由、绑定匹配、session key 和 lane key 构造。

#### `agent_gateway/runtime/execution/`

- `__init__.py`：execution 子包初始化文件。
- `agent_manifest.py`：Agent manifest 模板、校验和能力描述。
- `alerts_runtime.py`：告警运行时，基于指标和事件计算告警并触发通知。
- `autonomy.py`：主动任务运行时，包含 Heartbeat、Cron、Agent 局部 Cron、新闻简报等调度逻辑。
- `channel_runtime.py`：通道运行时，接收入站消息，处理 CLI 同步路径和非 CLI 持久化任务入队。
- `control_plane.py`：控制面对象，聚合配置、运行状态、事件、指标、任务、投递等查询和操作。
- `delivery_runtime.py`：可靠投递运行时，消费投递队列并调用具体通道发送消息。
- `dispatcher.py`：入站分发器，负责路由、调用 Agent 执行、生成回复并写入投递队列。
- `lanes.py`：单进程内命名并发车道和命令队列。
- `loop.py`：Agent Loop 执行闭环，组织单轮模型调用、工具调用和多轮交互。
- `metrics_runtime.py`：指标采集运行时，周期性采集队列、worker、投递、模型 profile 等状态。
- `resilience.py`：模型调用容错、profile 切换、tool calling 解析和 stop reason 处理。
- `roles.py`：运行角色展开逻辑，根据 `GATEWAY_RUNTIME_ROLES` 决定启动哪些 runtime。

#### `agent_gateway/runtime/infra/`

- `__init__.py`：infra 子包初始化文件。
- `postgres_client.py`：PostgreSQL 客户端封装。
- `rabbitmq.py`：RabbitMQ broker 封装，包含出站投递 broker 和入站任务 broker。
- `redis_client.py`：Redis 客户端封装，提供 ping、锁、去重、基础读写等能力。

#### `agent_gateway/runtime/observability/`

- `__init__.py`：observability 子包初始化文件。
- `alerts.py`：告警模型、告警规则和告警存储。
- `events.py`：统一运行事件模型、事件写入、读取、清洗和错误视图。
- `metrics.py`：指标快照模型、写入和读取逻辑。

#### `agent_gateway/runtime/state/`

- `__init__.py`：state 子包初始化文件。
- `adapter.py`：状态仓库适配层，用于把不同 store 接到统一 repository。
- `context.py`：长上下文保护和会话压缩逻辑。
- `factory.py`：状态仓库构造函数，根据配置创建 PostgreSQL、本地备份等读写 backend。
- `migration.py`：本地 JSONL / 文件状态向数据库迁移的辅助逻辑。
- `postgres.py`：PostgreSQL schema、读写仓库、任务 reserve、投递 reserve 和通用表操作。
- `queue.py`：可靠投递队列模型和本地投递状态管理。
- `repository.py`：状态读写接口定义。
- `store.py`：会话 JSONL 存储，负责会话历史追加、读取和重放。

#### `agent_gateway/runtime/tasks/`

- `__init__.py`：tasks 子包初始化文件和对外导出。
- `handlers.py`：后台任务 handler，尤其是 `agent_inbound` 的入站恢复、Redis lane lock、dispatcher 调用和回复投递。
- `lane.py`：Redis session lane 协调器，负责 owner、续租、释放、TTL 接管和状态镜像。
- `models.py`：`TaskInstance`、任务状态和序列化逻辑。
- `queue.py`：任务队列接口，负责 enqueue、reserve、ack、retry、fail、cancel、幂等查重，并在启用 session scheduler 时写入 Redis 调度索引。
- `session_scheduler.py`：Redis session ready scheduler，使用 per-session pending bucket、全局 ready index 和 busy owner，让同一 session 严格 FIFO，不同 session 可并行执行。
- `store.py`：任务本地文件存储，并可接入 PostgreSQL read/write backend。
- `worker.py`：后台任务 worker 池；启用 scheduler 时通过 Redis claim session 队首任务，执行期间续租 busy owner，RabbitMQ 作为削峰和唤醒；未启用时保留本地 reserve / broker task_id 兼容路径。

### 8.6 `workspace/reports/`

## 9. `data/`

本地运行数据目录。该目录通常会持续增长，主要用于运行时状态、审计、回放和兜底，不建议手工修改。

常见子目录：

- `sessions/`：会话历史 JSONL。
- `tasks/`：任务实例本地镜像。
- `delivery-queue/`：投递队列本地镜像。
- `events/`：运行事件 JSONL。
- `metrics/`：指标快照。
- `alerts/`：告警状态和历史。
- `feishu-webhook/`：飞书 Webhook 事件、审计和去重记录。
- `channel-state/`：通道侧状态。
- `onboarding/`：飞书绑定和 onboarding 状态。
- `news-digest/`：新闻简报采集和摘要产物。
- `github-skill-digest/`：GitHub Skill 推荐采集和去重状态。
- `migration/`：本地状态向数据库迁移时生成的中间记录和迁移审计。
