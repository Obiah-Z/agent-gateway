# 心跳任务说明

Heartbeat 是轻量后台检查，不是闲聊，也不是定时总结生成器。只有确实需要用户关注时才输出内容。

## 检查项

1. **到期提醒**：用户明确设置过、现在已经到期或即将到期的事项。
2. **风险跟进**：最近对话中出现过但尚未闭环的风险，例如失败任务、空间告警、投递异常、配置问题。
3. **用户明确要求持续关注的主题**：例如某个服务状态、某个项目进展、某类资料更新。
4. **每日总结**：仅当用户已经要求或系统配置需要时生成，不要默认每天硬发。

## 输出规则

- 没有需要用户关注的内容时，严格返回：`HEARTBEAT_OK`
- 有内容时直接给结果，不要写“我检查了”或“本次 heartbeat 检查发现”。
- 输出要短，包含：事项、原因、建议动作。
- 不要调用 `memory_write` 记录 heartbeat 过程。
- 不要为了显得主动而制造提醒。

## 优先级

提醒事项 > 系统风险 > 用户指定关注主题 > 每日总结
