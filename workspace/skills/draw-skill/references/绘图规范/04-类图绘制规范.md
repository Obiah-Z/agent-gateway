# 类图绘制规范

## 适用范围

类图用于表达面向对象设计中的类、接口、属性、操作和静态关系。适合论文中说明领域模型、设计模式、模块内部对象结构和接口契约。

## 权威依据

- OMG UML 2.5.1 Class Diagram / Classifier / Association / Generalization 相关规范。来源：https://www.omg.org/spec/UML/2.5.1/

## 基本元素

- 类：三栏矩形，类名、属性、操作。
- 接口：使用 `<<interface>>` 版型或棒棒糖接口符号。
- 抽象类/抽象操作：可使用斜体，或在名称中标注 `{abstract}`。
- 关联：实线，表示结构关系。
- 聚合：空心菱形端。
- 组合：实心菱形端，表示强生命周期从属。
- 泛化/继承：实线空心三角箭头指向父类。
- 实现：虚线空心三角箭头指向接口。
- 依赖：虚线开放箭头。
- 多重性：关系端标注 `1`、`0..1`、`*`、`1..*` 等。

## 绘制约束

1. 类名使用单数名词，接口名表达能力或角色。
2. 属性和操作只列与论文论证相关的核心项。
3. 可见性使用 UML 标记：`+` public，`-` private，`#` protected，`~` package。
4. 继承箭头必须指向父类；实现箭头必须指向接口。
5. 组合关系只在部分对象生命周期依赖整体对象时使用，不要滥用实心菱形。
6. 关联两端尽量写角色名或多重性，尤其是集合关系。
7. 不在类图中表达请求调用时序；动态调用应使用时序图。

## 论文 draw.io 样式

- 类框：白底黑边三栏矩形。
- 分隔线：黑色 1 px。
- 字体：宋体 12 pt；类名可加粗。
- 继承/实现箭头：空心三角，黑色；实现关系使用虚线。
- 聚合/组合菱形必须靠整体端。
- 大图可按包分组，但包边框仍为黑色细线。

## 常见错误

- 把数据库表当成类图，字段全是表字段。
- 继承箭头方向反了。
- 没有多重性，导致关系不明确。
- 使用组合表达普通引用。
- 类方法过多，图缩小后不可读。
