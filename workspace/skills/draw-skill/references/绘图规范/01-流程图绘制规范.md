# 流程图绘制规范

## 适用范围

流程图用于表达算法、程序控制流、数据处理步骤或业务处理步骤。论文中若强调“业务协作、角色、消息、异常事件”，优先使用 BPMN；若强调“程序/处理逻辑”，使用 ISO 5807 风格流程图。

## 权威依据

- ISO 5807:1985：定义 data flowchart、program flowchart、system flowchart、program network chart、system resources chart 的符号和约定。来源：https://standards.iteh.ai/catalog/standards/sist/e1a4e558-03f1-46cd-9ff7a9abc479294a/iso-5807-1985
- OMG BPMN 2.0.2：当图用于业务流程建模时，使用 BPMN 的事件、活动、网关、顺序流、消息流、泳池/泳道。来源：https://www.omg.org/spec/BPMN/2.0.2/

## 基本元素

ISO 5807 常用元素：

- 开始/结束：终端符，圆角矩形或胶囊形。
- 处理：矩形，表示处理函数、操作或步骤。
- 判断：菱形，单入口，多出口；出口线上标注条件。
- 数据/输入输出：平行四边形或 ISO 数据符号。
- 文档：底边波浪形文档符号。
- 预定义过程：双侧竖线矩形，表示子程序、模块或另处定义的过程。
- 流线：实线箭头，表示数据流或控制流。
- 连接符：页内小圆，跨页连接使用离页连接符。

BPMN 常用元素：

- 事件：圆形；开始事件细线，结束事件粗线，中间事件双线。
- 活动：圆角矩形。
- 网关：菱形，按语义使用排他、并行、事件等网关。
- 顺序流：实线箭头，只连接同一流程内的流对象。
- 消息流：虚线箭头，只用于不同参与者/泳池之间。
- 泳池/泳道：表示参与者或职责划分。

## 绘制约束

1. 流向默认自上而下；简单流程也可自左向右，但整图只能选择一种主方向。
2. 每个判断节点必须有明确出口标签，如“是/否”“满足/不满足”或条件表达式。
3. 避免一个处理节点表达多个动作。节点文字优先使用动宾短语。
4. 同一层级的步骤对齐，分支汇合点清晰。
5. 循环必须可识别：循环条件、返回边、退出边都要标注。
6. BPMN 中不要用普通流程线跨泳池，应使用消息流；同一泳池内部使用顺序流。
7. BPMN 不用于数据模型或组织结构模型，数据对象只作为活动输入/输出辅助说明。

## 论文 draw.io 样式

- 终端：白底黑边圆角矩形，`arcSize=50`。
- 处理：白底黑边矩形或轻微圆角矩形。
- 判断：白底黑边菱形。
- 线条：黑色，1 px，实线箭头。
- 字体：宋体 12 pt；判断节点内文字不超过两行。
- 不使用彩色填充。

## 常见错误

- 判断出口不写条件。
- 多个箭头交叉穿过节点。
- 把数据库表、接口、部署节点混入流程图。
- BPMN 中混用顺序流和消息流。
- 图中步骤粒度不一致，既有“完成订单”又有“校验手机号格式”。
