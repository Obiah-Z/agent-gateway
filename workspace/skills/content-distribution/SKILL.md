# 内容分发

当用户希望“一稿多发”、做社媒内容、项目日报/周报、发布公告、短视频脚本、Newsletter 或跨平台分发时，使用本技能。

## 核心原则

- 不同平台不能复制同一段话。
- 每个平台只承载一个清晰观点。
- 保留作者真实语气，不套平台模板。
- 不添加没有依据的 CTA、互动问题或情绪包装。
- 先从原始材料提炼事实，再改写成平台表达。

## 平台规则

### 飞书

- 简短、结构清晰、便于快速阅读。
- 适合用“结论 / 进展 / 风险 / 下一步”。

### X / Threads

- 直接给最强观点、反差或结果。
- Thread 每条都要推进信息，不要拆水文。

### LinkedIn

- 补充必要上下文，但不要写成假反思。
- 避免“很荣幸”“很激动”“旅程”等空泛表达。

### 公众号 / Newsletter

- 开头要交代读者能得到什么。
- 章节要有信息密度，不要长篇铺垫。

### 短视频

- 前 3 秒给结果、冲突或画面。
- 脚本围绕镜头和证据写，不写纸面好看但无法拍的旁白。

## 输出格式

返回：

- `核心角度`：一句话说明这次内容的主张。
- `平台版本`：分别给出每个平台的文本。
- `改写说明`：说明每个平台为什么这样改。
- `缺口`：发布前还需要补充的链接、截图、数据或素材。
