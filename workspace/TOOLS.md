# 工具使用说明

工具用于把推理结果落到真实环境中。调用工具前先判断：这个动作是否必要、是否安全、是否会改变外部状态。

## 常见工具

### 文件工具

- `read_file`：读取工作区内文件内容。
- `write_file`：写入文件内容，必要时创建父目录。
- `edit_file`：替换文件中的精确文本片段，`old_string` 必须唯一。
- `list_directory`：列出目录中的文件和子目录。

### 记忆工具

- `memory_write`：保存长期有价值的用户事实、偏好、项目决定和稳定背景。
- `memory_search`：按关键词检索既有记忆。

### 联网搜索工具

- `web_search`：搜索最新信息、外部事实、政策、价格、版本、公司或人物动态等可能变化的内容。
- `fetch_url`：读取指定 URL 正文，用于核验搜索结果中的关键来源。

### 系统工具

- `bash`：执行 shell 命令。优先只读检查，谨慎执行写入、删除、网络发布或长耗时命令。
- `get_current_time`：获取当前日期和时间。

## 调用原则

- 修改文件前先读取文件，除非用户明确给出完整新内容。
- 对当前时间敏感、外部变化频繁或高风险事实，先搜索或读取来源再回答。
- 工具输出只引用与任务相关的关键部分，避免把大量原始日志塞进回复。
- 搜索后尽量保留来源 URL；关键结论应使用 `fetch_url` 或等价方式核验。
- 工具失败时说明失败点和可替代路径，不要编造结果。

## 安全边界

- 不主动删除、覆盖、清空、推送、发布、付费、授权或修改凭据，除非用户明确要求并且目标范围清楚。
- 不把 API key、token、cookie、私钥等敏感信息写入普通回复、日志、记忆或文档。
- 对第三方系统的写操作要格外谨慎，例如 GitHub 推送、飞书发送、Webhook 配置、远程服务器命令。
- 如果 Runtime Context 标记某些工具被禁用，必须遵守，不要尝试绕过。

## 记忆写入边界

- 适合写入：用户长期偏好、稳定身份背景、项目长期技术决策、明确要求后续持续关注的事项。
- 不适合写入：Cron 输出、临时错误、一次性搜索结果、普通聊天摘要、工具调试过程。
- Cron、Heartbeat 等后台任务通常不应调用 `memory_write`；如果运行时禁用了该工具，必须严格遵守。
