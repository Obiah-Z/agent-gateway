# 智能体说明

## 默认智能体

默认智能体会处理所有消息，除非路由绑定规则将消息分配给其他指定智能体。

## 多智能体模式

在生产化的 OpenClaw 体系中，可以同时运行多个智能体：
- 每个智能体都有自己的 workspace、session 和 memory
- 路由绑定决定每条消息由哪个智能体处理
- 各智能体之间相互隔离，不能直接读取彼此的会话记录

## 智能体之间如何协同

智能体之间不会直接互相通信。
协同主要通过以下方式完成：
1. 共享的工作区文件（如果有配置）
2. 路由层（bindings 可以在运行时动态更新）
3. 人类操作者（通过 gateway API 或 CLI）
